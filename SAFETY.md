# SAFETY — 用前必读(招聘自动化是敏感领域)

这个 skill 操作**真实招聘账号**、触达**真实求职者**、消耗**真实付费权益**。下面每一条都是实战踩出来的,别跳过。

## 1. 永久红线(任何模式、任何档位都**不自动**,只在报告里给建议、等人来点)

- **换电话 / 换微信**(交换 PII)
- **约面**(对候选人的承诺)
- **删除职位**(**不可逆**破坏性)
- **举报 / 批量标记不合适**(破坏性 / 影响他人)

agent 只能把这些当"建议"写进报告,由人来点。

> **发布 / 关闭职位 ≠ 永久红线**(它们**可逆或可撤**,故 agent 可代操作,但有确认硬门):**关闭**可逆(进「已关闭」可重开、对话/简历保留),关前须核对是哪个岗 + 用户确认;**发布**对外公开且 4 字段(招聘类型/职位名称/职位类型/工作城市)发布后永久锁死,须**提交前回读这 4 项+内容给用户确认**、且发布后弹的**付费曝光推销只关不买**。细则见 SKILL.md 安全门 / operation-map §4.1–4.2。**唯一不可逆、永不自动的是「删除职位」。**

## 2. ⚠ 最大的坑:搜索畅聊卡"开聊" = 逐人消耗 1~3 张 + 自动索要 PII

搜索结果里点「搜索畅聊卡」开聊,**不是普通打招呼**:平台会一次性"发起沟通 **并索要简历/微信/电话**",内置了换微信/换电话的 PII 请求,还**逐人消耗 1~3 张**畅聊卡(读接口 `searchChatCardCostCount`,**非固定 3 张**,不是 1 张)。

- **用畅聊卡前必须让用户知情并接受这个捆绑**;别默认用户"授权畅聊卡"就等于接受自动索要 PII。
- 在 `strategy.yaml` 里显式授权:`budget.chat_cards > 0` **必须**同时设 `budget.authorize_card_pii_bundle: true`(知情接受 PII 捆绑的机检定义,validate.py 强制拦截);不接受就把 chat_cards 设 0。
- 预算**按逐人实际成本记账**(每次开聊 1~3 张,开聊前读 `searchChatCardCostCount` 得知该人成本);余量看 `geeks.json` 的 `chatCardCount`。别再按"固定 3 卡/次"估算。

## 3. 外发内容与消耗权益:按策略授权,别自作主张

- 打招呼 / 回复 / 求简历 = 外发内容;畅聊卡 / 道具 = 消耗权益。
- 策略寻访模式:按 `touch_policy`(`report_first` 零外发 / `greet_*` 自动打招呼 / `full` 自动打招呼 + 回复后求简历)和 `budget` 执行,逐张记账、超预算停。
- 单点操作模式:外发/消耗前逐条向用户确认。
- **`full` 档的"求简历"有平台前置门**:候选人未回复时按钮 disabled,必须对方回复(进"沟通中")才解锁——不是打招呼当下就能求。

## 3.5 发错人:收件人核验(真实事故换来的)

- 会话列表是**筛选页签作用域**的——切页签后目标可能不在可见列表,eval 找人会**静默失败**,当前会话停在上一个人;只核对消息内容不核对收件人,就会把 A 的消息发给 B(真实发生过)。
- **任何会话内外发(发消息/求简历)前必过三验**:①收件人(聊天头部 `.name-container .name-box` = 目标名)②内容 = 批准稿 ③发后回读(线程仍是目标)。细则见 operation-map §7c、SKILL.md 四条铁律。

## 4. 账号风控:低频、限速、分批

招聘方账号有行为风控,短时高频自动化会触发软限制(账号是公司资产)。

- 每日"主动沟通"有额度(实测企业版 **200/天**);留余量,别撞上限。
- **疑似风控信号**:聊天页反复卡"加载中"、额度异常掉零、动作被拒、要求验证 → **立即停手冷却**,别硬撞、别连环重试。
- **全局去重**:同一候选人 24h 内被多次触达(跨策略/跨同事)极易触发爬虫标记——触达前查会话列表比对,别重复 pitch。

## 5. 🔴 别乱重启 Chrome —— 会登出,且登出不可逆

- chrome-direct 接管的是你**真实登录的 Chrome**。**别 kill / 别在 0 窗口时 `--allow-restart-chrome`**——极易冲掉登录态。
- 一旦登出,Boss 要**手机微信扫码 / 短信验证码**重登,agent 代替不了、也不该碰验证码。
- 桥掉线先诊断(见 operation-map「桥掉线恢复姿势」),别赌重启。宁可停下让用户手动把 Boss 标签页点出来。

## 6. 数据保护:候选人 PII 别外泄

- 真跑产生的 `ledger.jsonl` 与 `reports/*.md` **含真实求职者姓名/公司/期望/联系方式** = 个人信息。**别提交到公开仓库、别外发。** 仓库自带的 `.gitignore` 已默认忽略这些,照它用。
- 求简历附件、截图同理(gitignore 已忽略 `*.doc/*.pdf/*.png` 等)。
- 别跨来源拼接候选人画像用于策略外的目的。

## 7. 凭据:agent 不碰

- 登录密码 / 验证码 / GitHub token / API key 等凭据,**agent 一律不代输、不在明文里处理**。需要认证的步骤(登录、git push)由用户在自己终端完成。
- browser-act API key 存在 CLI 本地配置里,别写进策略/报告/仓库。
